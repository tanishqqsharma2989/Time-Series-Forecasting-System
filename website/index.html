About The Project

What This Model Does

This project tackles the Kaggle Store Sales Time Series Forecasting competition — predicting unit sales for thousands of items sold at Corporación Favorita stores in Ecuador. The challenge involves modelling complex temporal dependencies across multiple stores, product families, and macro-economic signals like oil prices.

A CNN-BiLSTM (Convolutional + Bidirectional Long Short-Term Memory) hybrid architecture was designed and trained. The Conv1D layer extracts local temporal patterns, while stacked Bidirectional LSTM layers capture long-range dependencies in both directions across the time series.

External signals — oil price fluctuations, national holiday events, store metadata, and transaction volumes — are merged into a unified feature matrix, enabling the model to account for real-world economic context.

Methodology

Data Pipeline

End-to-end preprocessing transforms six raw CSV files into a clean, normalized feature matrix ready for deep learning.

Data Cleaning

Impute oil price missing values using backward-fill. Normalize transferred holiday records by removing "Translado" prefix and marking them as "Normal" type to avoid leakage.

fillna bfillstring replace
03

Dataset Merging

Sequentially merge train → holidays → oil → stores → transactions on date and store_nbr. Left joins preserve all training rows. Rename type_x/type_y for clarity.

left joinmerge on datemerge on store_nbr
04

Feature Engineering

Drop irrelevant columns (locale_name, description, transferred, city, state, cluster, store_type). Group by date and aggregate with mean to get one row per date.

groupbyagg meandrop columns
05

Categorical Encoding

Apply OneHotEncoder on the holiday_type column (Normal, Holiday, Event, Bridge, Work Day, Additional) producing 6 binary indicator features.

OneHotEncodersklearn

Normalization & Windowing

MinMaxScaler normalizes all features to [0, 1]. A series_to_supervised() function creates lag-1 input windows. Data is reshaped to (samples, 1, 10) for LSTM input.

MinMaxScalerlag-1reshape

Code Audit

Issues Found & Fixes Applied

A thorough review of cnn-bi-lstm-2.ipynb identified several issues. All have been documented and patched.

Medium

Undefined variable model used instead of model1

In the train-prediction cell, model.predict(X_train) references an undefined model variable — the correct reference is model1. This causes a NameError at runtime.

Fix: train_predict = model1.predict(X_train)
Medium

Incorrect inverse transform on train predictions

train_predict = scaler.inverse_transform(X_train) overwrites the prediction with the raw training features. Should concatenate the prediction with remaining features first.

Fix: train_predict = np.concatenate((train_predict, X_train[:, 1:]), axis=1)
train_predict = scaler.inverse_transform(train_predict)[:, 0]
Low

Deprecated fillna(method='bfill') API

Using fillna(method='bfill') is deprecated in pandas ≥ 2.x and will raise a FutureWarning or error in future versions.

Fix: oil["dcoilwtico"].bfill(inplace=True) df["dcoilwtico"] = df["dcoilwtico"].bfill()
Low

Deprecated OneHotEncoder(sparse=False)

The sparse parameter was renamed to sparse_output in scikit-learn ≥ 1.2 and will raise a TypeError in future versions.

Fix: OneHotEncoder(handle_unknown="ignore", sparse_output=False)
Info

Kaggle-specific hardcoded data paths prevent local execution

All data paths point to /kaggle/input/store-sales.../ — the notebook will crash when run outside Kaggle. A local path fallback improves portability.

Fix: BASE = "/kaggle/input/store-sales-time-series-forecasting/" if os.path.exists("/kaggle") else "../Dataset/"
Info

Untitled.ipynb is an empty scratch file

The second notebook contains only empty cells and an import error — it should be deleted before GitHub submission to keep the repo clean.

Fix: Delete Untitled.ipynb or add it to .gitignore
